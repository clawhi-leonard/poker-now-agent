## Critical Issue: Slider Reliability

**This is the #1 blocker for sustainable multi-hand sessions.**

The pokernow raise slider is not reliably set by JS. Current approach:
1. `_set_bet_amount()` sets text input + range slider via React nativeSetter
2. Falls back to Playwright keyboard typing (triple-click, Cmd+A, type number)
3. Multiple retry attempts

**What happens:** The slider snaps to max instead of the target value. This causes:
- Bots go all-in unexpectedly
- Mass bust-outs every 4-5 hands
- Most of the session time is spent in rebuy cycles

**Potential fixes:**
1. **Slider position calculation:** Instead of setting value, calculate pixel offset and drag
2. **Min-bet fallback:** If slider fails, just click the "BET 20" preset button
3. **Call-only mode:** When slider is unreliable, just call instead of raising
4. **Keyboard arrow keys:** Use left/right arrows on the slider to decrement from max

## Session Architecture Issues
