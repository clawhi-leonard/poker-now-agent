## Architecture Notes

### Why Tier 2 (Arrow Keys) Dominates
- Arrow keys directly interact with the browser's native range input handling
- React receives proper `input` events from native keyboard interaction
- No need to hack around React's synthetic event system
- Works reliably even when JS value setters fail

### Why Tier 1 (Mouse) Also Works
- Mouse click on slider track position works for larger value changes
- Less precise than arrow keys (within ~10% of target vs exact)
- Better for large jumps (setting to 1000 when range is 0-5000)
