## Issues Observed

### 1. Re-raise Escalation (Still Present)
Both players frequently escalate to all-in on flop:
```
NitKing flop raise 44 → CallStn raise 132 → NitKing raise 220 → CallStn raise 848 → NitKing raise 980 → all-in
```
Both showing ~85% equity against each other — this can happen when both have strong hands, but the bot should cap re-raises (max 3-4 raises per street to avoid leveling wars).

### 2. Passive Postflop Heads-Up
When both have medium hands (40-55% equity), they often check-check through all streets:
```
AceBot river 5h8h eq=35% → check | NitKing river 7h4h eq=36% → check
```
This is reasonable for weak hands, but more thin value betting should happen.

### 3. Equity Monte Carlo Variance
Seeing both players show 80%+ equity against each other — statistically impossible. The MC sim randomizes opponent hands independently per bot, so each calculates equity against random hands, not against the actual opponent.

### 4. Cookie Banner Interference 
`<div class="alert-1-container">…</div> intercepts pointer events` errors appearing — the dismiss_cookie_banner isn't always removing it from DOM before Playwright clicks. Should be more aggressive about removal.
