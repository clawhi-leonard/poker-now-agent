## Key Issues Found

### 1. Broken Raise Slider (12 occurrences)
When raising, 12 times the raise showed "Raised to ? (range 2-100)" instead of the intended amount. The slider wasn't set properly. This is ~39% of all raises.
- **Root Cause:** The act.py `execute_action()` may not be handling the slider/input correctly for bet amounts.
- **Impact:** High — bots sometimes bet min or max when they intended something else.
- **Fix Priority:** HIGH

### 2. No Auto-Rebuy
AceBot and CallStn busted and never rebuyed. The `auto_rebuy()` function exists but wasn't triggered effectively.
- **Root Cause:** The rebuy check only looks for 0 stack, but doesn't detect when a player has been eliminated from the game entirely.
- **Impact:** High — game effectively ended when 2 bots busted.
- **Fix Priority:** HIGH

### 3. Fold Rate Too Low
Only 14% of actions were folds. In a 4-handed game, bots should fold ~40-50% preflop.
- **Root Cause:** The tightness thresholds are too low. NIT with 60 tightness should fold more but is calling with 41% equity hands heads-up.
- **Impact:** Medium — leads to calling with weak hands.

### 4. Duplicate/Stale Actions
Some hands show the same bot acting multiple times in sequence on the same street (e.g., Clawhi acting twice preflop). The 2-second cooldown helped but didn't eliminate this.
- **Impact:** Low-Medium — occasionally wastes actions but doesn't crash.
